# Codex Reminders

Project: Multi Sale Organizer
Profile: webapp

## Trước khi Codex viết code

- Đọc `MATRIX_HANDOFF.md` để biết việc cần làm tiếp.
- Đọc `docs/ARCHITECTURE.md` để không phá ranh giới module.
- Đọc `docs/MODULE_MAP.md` để đặt file đúng nơi.
- Đọc `docs/UX_UI_FLOW.md` trước khi viết route, màn hình, component hoặc interaction.
- Đọc `docs/TECH_RADAR.md` trước khi thêm framework, thư viện, SDK, API hoặc dịch vụ mới.
- Đọc `docs/LAUNCH_READINESS.md`, `docs/COMMERCIAL_READINESS.md` và `docs/OPERATIONS_RUNBOOK.md` trước khi nói dự án sẵn sàng online/thương mại.
- Đọc `docs/SKILL_FACTORY.md` trước khi tạo hoặc sửa skill cho dự án.
- Chạy hoặc đề xuất `quality-gate` nếu cấu trúc dự án vừa thay đổi.

## Khi có công nghệ mới

- Nếu thông tin có thể đã thay đổi, Codex phải tra cứu/xác minh trước khi kết luận.
- Ghi công nghệ vào `docs/TECH_RADAR.md` với trạng thái `adopt`, `trial`, `assess`, hoặc `hold`.
- Nếu đổi stack, thêm ADR mới trong `docs/adr/`.
- Nếu công nghệ mới ảnh hưởng nhiều plugin, cập nhật `MATRIX_PLUGIN_MAP.md`.

## Khi bàn giao kết quả

- Nói rõ đã làm gì.
- Nói rõ có lỗi hoặc rủi ro không.
- Nói rõ phần nào cần người dùng kiểm tra trước khi triển khai tiếp.
